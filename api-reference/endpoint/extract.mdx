---
title: Extract
openapi: POST /extract
---

Upload one or more CRE documents (rent rolls, T12s, operating statements, offering memorandums) and receive structured extraction results. Supported formats include PDF, XLSX, XLS, and CSV files.

The response contains four top-level sections:

- **`property`** — Extracted property details such as name, address, purchase price, year built, and unit count.
- **`financials`** — Income and expense line items extracted from operating statements and T12s.
- **`rent_stack`** — Unit-level rent data parsed from rent rolls, broken down by unit type.
- **`documents`** — Metadata for each uploaded file, including its detected type and extraction status.

## Response structure

### Financials

The `financials` object contains `mapped_income` and `mapped_expenses`, each returned as a flat array of `FinancialNode` objects. Every node represents a single line item from the source document:

| Field | Type | Description |
|-------|------|-------------|
| `id` | integer | Row index from the source document |
| `field` | string | Standardized category slug (e.g. `gross_potential_rent`, `tax_real_property`) |
| `display_name` | string | Original label as it appears in the document |
| `value` | string | Dollar amount as a string (e.g. `"864000.00"`) |
| `citation` | string | Source reference (e.g. `"row:5"`) |
| `user_adjusted` | boolean | Whether this value has been manually adjusted |
| `source_items` | integer[] | IDs of child line items that roll up into this node |

Subtotal rows use `source_items` to reference their children. For example, a "Utilities" node with `"source_items": [11, 12]` is the sum of the nodes with `id` 11 and 12 (e.g. "Water & Sewer" and "Electric - Common Areas"). Leaf nodes have an empty `source_items` array.

### Rent stack

Each item in the `rent_stack` array represents one unit type with its count, square footage, and rent figures (current, market, and projected).

<RequestExample>
```bash cURL
curl -X POST https://api.runmilo.ai/v1/extract \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -F "files=@rent-roll.pdf" \
  -F "files=@operating-statement.xlsx"
```

```python Python
import requests

response = requests.post(
    "https://api.runmilo.ai/v1/extract",
    headers={"Authorization": "Bearer YOUR_API_KEY"},
    files=[
        ("files", open("rent-roll.pdf", "rb")),
        ("files", open("operating-statement.xlsx", "rb")),
    ],
)

print(response.json())
```

```javascript Node.js
const form = new FormData();
form.append("files", fs.createReadStream("rent-roll.pdf"));
form.append("files", fs.createReadStream("operating-statement.xlsx"));

const response = await fetch("https://api.runmilo.ai/v1/extract", {
  method: "POST",
  headers: { Authorization: "Bearer YOUR_API_KEY" },
  body: form,
});

const data = await response.json();
```
